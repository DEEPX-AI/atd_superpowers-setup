로그인 API 구현 접근 방식

Node.js + Express + JWT + SQLite 기반 — 어떤 구조로 구현할까요?

A

미니멀 플랫 구조

라우터 파일에 비즈니스 로직을 직접 작성. 빠르고 간단하지만 확장성이 낮음.

  • 파일 수 최소화 (3~4개)
  • 작은 프로젝트에 적합
  • 테스트 작성이 어려움
B

레이어드 아키텍처 (추천)

Route → Controller → Service → Repository 계층 분리. 관심사가 명확하고 테스트 친화적.

  • 각 계층이 단일 책임을 가짐
  • 서비스 로직을 독립적으로 테스트 가능
  • Express → Fastify 마이그레이션도 용이
C

Passport.js 기반

passport-local 전략 사용. 추후 OAuth 확장이 쉽지만 현재 요구사항 대비 오버엔지니어링.

  • OAuth2 / 소셜 로그인 추가 시 유리
  • 추상화 레이어가 추가됨
  • JWT only 용도엔 다소 무거움
